# Git Branch Policy

**Effective Immediately**: All development work must be done on preview branches, NOT on `main`.

## Branch Strategy

### Production Branch
- **Branch**: `main`
- **Purpose**: Production-ready code only
- **Deployment**: Auto-deploys to production (Vercel)
- **Access**: **RESTRICTED** - No direct commits allowed

### Preview Branches
- **Naming**: `preview/*` (e.g., `preview/redevelopment`, `preview/feature-name`)
- **Purpose**: Development, testing, and staging
- **Deployment**: Auto-deploys to Vercel preview environment
- **Access**: **OPEN** - All development work happens here

## Workflow

1. **Create preview branch from main**:
   ```bash
   git checkout main
   git pull origin main
   git checkout -b preview/feature-name
   ```

2. **Develop and commit on preview branch**:
   ```bash
   git add .
   git commit -m "feat: your changes"
   git push origin preview/feature-name
   ```

3. **Test on Vercel preview deployment**

4. **Merge to main only after approval**:
   ```bash
   git checkout main
   git merge preview/feature-name
   git push origin main
   ```
